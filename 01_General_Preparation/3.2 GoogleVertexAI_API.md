# 🔷 Configuration & Connection Guide: Google Vertex AI

This guide covers project setup, authentication, credential configuration, and basic integration for **Google Vertex AI** — following the same structure as the OpenAI and Azure guides.

> ⚠️ **Key difference from other providers:** Vertex AI does NOT use simple API keys for production use. Authentication is handled through **Google Cloud IAM** — either Application Default Credentials (ADC), a Service Account JSON key, or a Cloud API key. Choose the method that matches your environment.

---

## 1️⃣ Create Your Google Cloud Project & Enable Vertex AI

1. Go to the [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project or select an existing one — note your **Project ID**
3. In the console, navigate to **APIs & Services → Enable APIs**
4. Search for **"Vertex AI API"** and click **Enable**
5. Navigate to **IAM & Admin → IAM** and ensure your user or service account has the **`Vertex AI User`** role (`roles/aiplatform.user`)

> 📌 Note: **AI Studio API keys (`AIzaSy...`) do NOT work with Vertex AI.** These are separate systems. Vertex AI requires GCP IAM authentication.

---

## 2️⃣ Choose Your Authentication Method

Vertex AI supports three auth methods. Pick the one that fits your environment:

### Method A — Application Default Credentials (ADC) via gcloud CLI
**Best for:** Local development on your own machine.

```bash
# Install gcloud CLI: https://cloud.google.com/sdk/docs/install
gcloud auth application-default login
```

This opens a browser, logs you in, and saves credentials to `~/.config/gcloud/application_default_credentials.json`. The SDK picks them up automatically.

Set env vars:

```env
GOOGLE_CLOUD_PROJECT="your-project-id"
GOOGLE_CLOUD_LOCATION="us-central1"
GOOGLE_GENAI_USE_VERTEXAI=true
```

---

### Method B — Service Account JSON Key
**Best for:** CI/CD pipelines, Docker containers, restricted environments where gcloud CLI cannot be installed.

1. In the console, go to **IAM & Admin → Service Accounts**
2. Click **"Create Service Account"**, give it a name
3. Assign the **`Vertex AI User`** role
4. Under the service account, go to **Keys → Add Key → JSON** — download the file
5. Store it securely (never commit to version control)

Set env vars:

```env
GOOGLE_APPLICATION_CREDENTIALS="/absolute/path/to/your-service-account-key.json"
GOOGLE_CLOUD_PROJECT="your-project-id"
GOOGLE_CLOUD_LOCATION="us-central1"
GOOGLE_GENAI_USE_VERTEXAI=true
```

---

### Method C — Cloud API Key (simplest, limited)
**Best for:** Quick tests only. Some organizations restrict this.

1. In the console, go to **APIs & Services → Credentials → Create Credentials → API Key**
2. Restrict the key to the **Vertex AI API**

```env
GOOGLE_API_KEY="your-cloud-api-key"
GOOGLE_CLOUD_PROJECT="your-project-id"
GOOGLE_CLOUD_LOCATION="us-central1"
GOOGLE_GENAI_USE_VERTEXAI=true
```

> ⚠️ If you see `"API keys are not supported by this API"`, your organization has restricted API key usage. Use Method A or B instead.

---

## 3️⃣ Add Billing

Vertex AI is billed through your Google Cloud account — no separate payment method beyond what's already linked to your GCP project.

1. In the console, go to **Billing** and link a billing account to your project
2. New GCP accounts receive **$300 in free credits** valid for 90 days
3. After credits, pay-as-you-go per token

> 📊 Track usage in [Google Cloud Billing](https://console.cloud.google.com/billing) and set **Budget Alerts** to avoid unexpected charges.

## 5️⃣ Test Your Credentials

Create a simple script (`test_vertexai.py`) — works with Method A (ADC) or Method B (service account via `GOOGLE_APPLICATION_CREDENTIALS`):

```python
import os
from google import genai
from dotenv import load_dotenv

load_dotenv()

# With GOOGLE_GENAI_USE_VERTEXAI=true set, the client auto-uses Vertex AI
client = genai.Client(
    vertexai=True,
    project=os.getenv("GOOGLE_CLOUD_PROJECT"),
    location=os.getenv("GOOGLE_CLOUD_LOCATION", "us-central1")
)

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="Write a short story about the Roman Empire."
)

print(response.text)
```

👉 If you see a short story printed, your setup is correct!

### A. Raw API Test Script (with error handling + token tracking)

```python
import os
from google import genai
from google.genai import types
from dotenv import load_dotenv

load_dotenv()

client = genai.Client(
    vertexai=True,
    project=os.getenv("GOOGLE_CLOUD_PROJECT"),
    location=os.getenv("GOOGLE_CLOUD_LOCATION", "us-central1")
)

try:
    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents="Write a short story about the Roman Empire.",
        config=types.GenerateContentConfig(
            system_instruction="You are a helpful assistant.",
            max_output_tokens=200,
            temperature=0.7,
        )
    )
    print(f"✅ Connection Successful:\n{response.text}")
    print(f"\n📊 Token Usage → Input: {response.usage_metadata.prompt_token_count} | Output: {response.usage_metadata.candidates_token_count}")

except Exception as e:
    print(f"❌ Error: {e}")
    if "403" in str(e) or "PERMISSION_DENIED" in str(e):
        print("   → Check that your account has the 'Vertex AI User' IAM role on this project")
    if "credentials" in str(e).lower():
        print("   → Run 'gcloud auth application-default login' or check GOOGLE_APPLICATION_CREDENTIALS")
```

### B. Service Account Key — Explicit Auth (no env vars required)

If you prefer to load the service account file explicitly in code rather than via `GOOGLE_APPLICATION_CREDENTIALS`:

```python
import vertexai
from vertexai.generative_models import GenerativeModel
from google.oauth2 import service_account

credentials = service_account.Credentials.from_service_account_file(
    "path/to/your-service-account-key.json",
    scopes=["https://www.googleapis.com/auth/cloud-platform"]
)

vertexai.init(
    project="your-project-id",
    location="us-central1",
    credentials=credentials
)

model = GenerativeModel("gemini-2.5-flash")
response = model.generate_content("Write a short story about the Roman Empire.")
print(response.text)
```

### C. OpenAI-Compatible Endpoint (drop-in for OpenAI SDK users)

Vertex AI exposes an OpenAI-compatible endpoint. Auth uses a short-lived OAuth token instead of an API key:

```python
import os
import google.auth
import google.auth.transport.requests
from openai import OpenAI

project_id = os.getenv("GOOGLE_CLOUD_PROJECT")
location   = os.getenv("GOOGLE_CLOUD_LOCATION", "us-central1")

# Get a short-lived OAuth token from ADC
credentials, _ = google.auth.default(scopes=["https://www.googleapis.com/auth/cloud-platform"])
credentials.refresh(google.auth.transport.requests.Request())

client = OpenAI(
    api_key=credentials.token,   # OAuth token, not an API key
    base_url=f"https://{location}-aiplatform.googleapis.com/v1/projects/{project_id}/locations/{location}/endpoints/openapi"
)

response = client.chat.completions.create(
    model="google/gemini-2.5-flash",
    messages=[{"role": "user", "content": "Write a short story about the Roman Empire."}],
    max_tokens=200
)
print(response.choices[0].message.content)
```

> ⚠️ OAuth tokens expire after **1 hour**. For long-running applications, refresh the token periodically and update `client.api_key`.

## 🔍 Tips & Best Practices

- **Never commit service account JSON key files to version control** — treat them like passwords
- For workloads running on GCP (Cloud Run, GKE, Compute Engine), use **Workload Identity / attached service accounts** — no key files needed at all
- **ADC (Method A) for local dev, service accounts (Method B) for CI/CD** — this is the standard GCP pattern
- The `GOOGLE_GENAI_USE_VERTEXAI=true` env variable is the simplest way to switch the `google-genai` SDK from AI Studio to Vertex AI without code changes
- Set **GCP Budget Alerts** — Vertex AI has no built-in spend cap
- Token usage is tracked per-project in [Google Cloud Billing](https://console.cloud.google.com/billing)
- If you get `403 PERMISSION_DENIED`, verify the IAM role (`Vertex AI User`) is assigned to the correct identity (your user or service account) on the correct project
