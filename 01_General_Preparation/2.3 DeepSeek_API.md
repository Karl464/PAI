# 🔬 Configuration & Connection Guide: DeepSeek API

This guide covers account creation, API key generation, credential validation, and basic integration for the **DeepSeek API** — following the same structure as the OpenAI and Azure guides.

---

## 1️⃣ Create Your DeepSeek Account & API Key

1. Go to [platform.deepseek.com](https://platform.deepseek.com/)
2. Sign up with email (no Chinese phone number required for the API platform)
3. Navigate to **API Keys** in the dashboard
4. Click **"Create new API key"**, give it a descriptive name, and copy it immediately

> ⚠️ The key is shown only once. If you close without copying, you must generate a new one.

Save the key in your `.env` file:

```env
DEEPSEEK_API_KEY="sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
DEEPSEEK_MODEL="deepseek-v4-flash"
DEEPSEEK_BASE_URL="https://api.deepseek.com"
```

> 🔑 DeepSeek API keys start with `sk-`. Note: the old aliases `deepseek-chat` and `deepseek-reasoner` are deprecated and will be removed on **2026-07-24**. Use `deepseek-v4-flash` or `deepseek-v4-pro` directly.

---

## 2️⃣ Add Billing & Free Credits

New accounts receive **5 million free tokens** — no credit card required.

For continued usage after free credits:

1. In the platform, navigate to **Billing** and add a minimum top-up of **$2**
2. The API requires a balance before processing paid requests
3. Usage is deducted from your granted balance first, then your topped-up balance

> 📊 Track consumption at [platform.deepseek.com](https://platform.deepseek.com/) under the Usage section. You can also check your balance via the `/user/balance` API endpoint.

**Off-peak discount window:** DeepSeek historically offers 50–75% discounts during **16:30 to 00:30 UTC**. Scheduling batch jobs in this window can significantly reduce costs.

## 4️⃣ Test Your Credentials

Create a simple script (`test_deepseek.py`) to confirm the connection works:

```python
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()

client = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL", "https://api.deepseek.com")
)

response = client.chat.completions.create(
    model=os.getenv("DEEPSEEK_MODEL", "deepseek-v4-flash"),
    messages=[{"role": "user", "content": "Write a short story about the Roman Empire."}],
    max_tokens=200
)

print(response.choices[0].message.content)
```

👉 If you see a short story printed, your setup is correct!

### A. Raw API Test Script (with error handling + cache tracking)

```python
import os
from openai import OpenAI, APIStatusError, APIConnectionError
from dotenv import load_dotenv

load_dotenv()

client = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL", "https://api.deepseek.com")
)

try:
    response = client.chat.completions.create(
        model=os.getenv("DEEPSEEK_MODEL", "deepseek-v4-flash"),
        messages=[
            {"role": "system", "content": "You are a helpful assistant."},
            {"role": "user", "content": "Write a short story about the Roman Empire."}
        ],
        max_tokens=200,
        temperature=0.7,
    )
    print(f"✅ Connection Successful:\n{response.choices[0].message.content}")

    usage = response.usage
    cache_hit    = getattr(usage, "prompt_cache_hit_tokens", 0)
    cache_miss   = getattr(usage, "prompt_cache_miss_tokens", 0)
    output_tokens = usage.completion_tokens

    print(f"\n📊 Token Usage → Cache Hit: {cache_hit} | Cache Miss: {cache_miss} | Output: {output_tokens}")

except APIStatusError as e:
    print(f"❌ API Error {e.status_code}: {e.message}")
except APIConnectionError as e:
    print(f"❌ Connection Error: {e}")
```

> **Note:** DeepSeek splits input tokens into `prompt_cache_hit_tokens` and `prompt_cache_miss_tokens` in the usage object. Cache hits are billed at up to 98% less than cache misses — tracking this ratio is key to cost optimization.

### B. Enabling Thinking Mode (Chain-of-Thought)

For complex reasoning tasks, enable thinking mode on `deepseek-v4-pro`:

```python
response = client.chat.completions.create(
    model="deepseek-v4-pro",
    messages=[
        {"role": "user", "content": "Solve this step by step: if a train travels 120km at 60km/h, then 80km at 40km/h, what is the average speed?"}
    ],
    extra_body={
        "thinking": {"type": "enabled"},
        "reasoning_effort": "high"   # "low", "medium", or "high"
    }
)
print(response.choices[0].message.content)
```

## 5️⃣ Available Models & Pricing (May 2026)

All prices are per **million tokens (MTok)**. DeepSeek splits input billing into cache hit and cache miss buckets.

| Model | String | Cache Miss Input ($/MTok) | Cache Hit Input ($/MTok) | Output ($/MTok) | Best For |
|-------|--------|--------------------------|--------------------------|-----------------|----------|
| V4 Flash | `deepseek-v4-flash` | $0.14 | $0.0028 | $0.28 | Default — cost leader, 1M context |
| V4 Pro | `deepseek-v4-pro` | $0.435* | $0.003625* | $0.87* | Advanced reasoning, agentic tasks |

> *V4 Pro is at **75% promotional discount** until 2026-05-31 15:59 UTC. Standard rates are $1.74/$0.01/$3.48.

**Context window:** Both V4 models support **1 million tokens** input and up to **384K tokens** output.

**Cache pricing is automatic** — no configuration needed. DeepSeek caches repeated prompt prefixes automatically. Front-load static content (system prompts, document templates) before variable content to maximize cache hits.

## 🔍 Tips & Best Practices

- Keep your `.env` private — **never commit `sk-` DeepSeek keys to version control**
- **Use `deepseek-v4-flash`** for most tasks — it's the cost leader and handles the vast majority of workloads
- **Use `deepseek-v4-pro`** only when you need deep multi-step reasoning or complex agentic tasks
- Front-load static content in prompts to maximize the 98% cache hit discount
- Schedule batch jobs during the off-peak window (16:30–00:30 UTC) for additional savings
- Track cached vs. non-cached tokens in `response.usage` to understand your effective cost per request
- If you get `429 Too Many Requests`, DeepSeek has experienced intermittent high-demand outages — implement retry with exponential backoff
