# 🇫🇷 Configuration & Connection Guide: Mistral AI API

This guide covers account creation, API key generation, credential validation, and basic integration for the **Mistral AI API (La Plateforme)** — following the same structure as the OpenAI and Azure guides.

---

## 1️⃣ Create Your Mistral Account & API Key

1. Go to [console.mistral.ai](https://console.mistral.ai/)
2. Sign up with email or Google SSO
3. Navigate to **API Keys** in the left sidebar
4. Click **"Create new key"**, optionally scope it to a specific project, and copy it immediately

> ⚠️ The key is shown only once. If you close without copying, you must generate a new one.

Save the key in your `.env` file:

```env
MISTRAL_API_KEY="xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
MISTRAL_MODEL="mistral-large-latest"
```

> 🔑 Mistral API keys are alphanumeric strings without a fixed prefix. Keys can be **scoped to projects** to separate billing across teams — recommended for multi-project setups.

> ⚠️ **Consumer vs API billing are separate.** A Le Chat Pro subscription ($14.99/month) does NOT include API credits. The API is billed independently per token through La Plateforme.

---

## 2️⃣ Add Billing

Mistral's API includes a **free experiment tier** with rate limits — suitable for testing and prototyping, not production.

For production workloads:

1. In [console.mistral.ai](https://console.mistral.ai/), navigate to **Billing**
2. Add a payment method
3. Mistral bills **monthly** based on accumulated usage

> 📊 Track usage and costs at [console.mistral.ai](https://console.mistral.ai/) under the Usage section.

**Enterprise / committed-use pricing:** Available for teams with predictable high volume via direct sales at [mistral.ai/enterprise](https://mistral.ai/enterprise).

## 4️⃣ Test Your Credentials

### Using the native Mistral SDK

Create a simple script (`test_mistral.py`) to confirm the connection works:

```python
import os
from mistralai import Mistral
from dotenv import load_dotenv

load_dotenv()

client = Mistral(api_key=os.getenv("MISTRAL_API_KEY"))

response = client.chat.complete(
    model=os.getenv("MISTRAL_MODEL", "mistral-large-latest"),
    messages=[{"role": "user", "content": "Write a short story about the Roman Empire."}]
)

print(response.choices[0].message.content)
```

👉 If you see a short story printed, your setup is correct!

### A. Raw API Test Script (with error handling)

```python
import os
from mistralai import Mistral
from mistralai.models import SDKError
from dotenv import load_dotenv

load_dotenv()

client = Mistral(api_key=os.getenv("MISTRAL_API_KEY"))

try:
    response = client.chat.complete(
        model=os.getenv("MISTRAL_MODEL", "mistral-large-latest"),
        messages=[
            {"role": "system", "content": "You are a helpful assistant."},
            {"role": "user", "content": "Write a short story about the Roman Empire."}
        ],
        max_tokens=200,
        temperature=0.7,
    )
    print(f"✅ Connection Successful:\n{response.choices[0].message.content}")
    print(f"\n📊 Token Usage → Input: {response.usage.prompt_tokens} | Output: {response.usage.completion_tokens}")

except SDKError as e:
    print(f"❌ Mistral SDK Error: {e}")
except Exception as e:
    print(f"❌ Error: {e}")
```

### B. OpenAI SDK Alternative (drop-in replacement)

If you already use the OpenAI SDK, switching to Mistral is a two-line change:

```python
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()

client = OpenAI(
    api_key=os.getenv("MISTRAL_API_KEY"),
    base_url="https://api.mistral.ai/v1"
)

try:
    response = client.chat.completions.create(
        model=os.getenv("MISTRAL_MODEL", "mistral-large-latest"),
        messages=[
            {"role": "system", "content": "You are a helpful assistant."},
            {"role": "user", "content": "Write a short story about the Roman Empire."}
        ],
        max_tokens=200,
        temperature=0.7,
    )
    print(f"✅ Connection Successful:\n{response.choices[0].message.content}")
except Exception as e:
    print(f"❌ Error: {e}")
```

### C. Code Generation with Codestral (Fill-in-the-Middle)

Codestral's unique fill-in-the-middle feature understands code before AND after the cursor:

```python
from mistralai import Mistral
import os

client = Mistral(api_key=os.getenv("MISTRAL_API_KEY"))

response = client.fim.complete(
    model="codestral-latest",
    prompt="def fibonacci(n):\n    ",
    suffix="\n    return result"
)
print(response.choices[0].message.content)
```

## 5️⃣ Available Models & Pricing (May 2026)

All prices are per **million tokens (MTok)**, billed separately for input and output.

| Model | String | Input ($/MTok) | Output ($/MTok) | Best For |
|-------|--------|----------------|-----------------|----------|
| Mistral Large 2 | `mistral-large-latest` | $2.00 | $6.00 | Complex reasoning, multilingual |
| Mistral Medium 3 | `mistral-medium-latest` | $1.00 | $3.00 | Balanced general-purpose tasks |
| Mistral Small 3.1 | `mistral-small-latest` | $0.10–$0.20 | $0.30–$0.60 | High-volume, cost-sensitive tasks |
| Codestral | `codestral-latest` | $0.30 | $0.90 | Code generation, fill-in-middle |
| Devstral 2 | `devstral-2-latest` | $0.40 | $2.00 | Agentic coding, CI/CD automation |
| Ministral 3B | `ministral-3b-latest` | $0.04 | $0.04 | Edge deployment, on-device inference |

> ⚠️ **No prompt caching as of May 2026.** Unlike Anthropic and OpenAI, Mistral does not offer cached-read discounts. Factor this into cost estimates for workloads with large repeated system prompts.

**Key pricing advantage:** Mistral Large 2 at $6.00/M output is significantly cheaper than comparable frontier models — important for long-form generation (code, reports, analysis).

**Self-hosting option:** Several Mistral models (Mistral 7B, Mixtral 8x7B, Mixtral 8x22B) are available under Apache 2.0 for on-premise or air-gapped deployments with zero API costs.

## 🔍 Tips & Best Practices

- Keep your `.env` private — **never commit Mistral API keys to version control**
- **Scope keys to projects** in the console to isolate billing across environments (dev / staging / prod)
- Use `mistral-small-latest` for 80%+ of production chat requests — it handles most tasks adequately
- Use `mistral-large-latest` or `mistral-medium-latest` only for complex reasoning or high-accuracy requirements
- **EU data residency by default** — all inference runs on European infrastructure, GDPR-compliant with no extra configuration
- For coding workflows, `codestral-latest` or `devstral-2-latest` outperform general-purpose models on code tasks
- On `429 Too Many Requests`, implement exponential backoff — or consider committed-use pricing for consistent high-volume workloads
